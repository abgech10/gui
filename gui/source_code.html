from PySide6.QtWidgets import QApplication, QWidget, QVBoxLayout, QLabel, QLineEdit, QPushButton, QTextEdit
import sys

# Student Class (Encapsulation)
class Student:
    def __init__(self, name, age, course):
        self.name = name
        self.age = age
        self.course = course

    def get_details(self):
        return f"Name: {self.name}, Age: {self.age}, Course: {self.course}"

# Main App Class (Inheritance from QWidget)
class StudentApp(QWidget):
    def __init__(self):
        super().__init__()
        self.setWindowTitle("Student Registration System")
        self.setGeometry(200, 200, 400, 300)
        
        self.students = []  # List to store student objects

        # UI Elements
        self.label_name = QLabel("Name:")
        self.input_name = QLineEdit()

        self.label_age = QLabel("Age:")
        self.input_age = QLineEdit()

        self.label_course = QLabel("Course:")
        self.input_course = QLineEdit()

        self.register_button = QPushButton("Register Student")
        self.register_button.clicked.connect(self.register_student)

        self.result_display = QTextEdit()
        self.result_display.setReadOnly(True)

        # Layout (Vertical)
        layout = QVBoxLayout()
        layout.addWidget(self.label_name)
        layout.addWidget(self.input_name)
        layout.addWidget(self.label_age)
        layout.addWidget(self.input_age)
        layout.addWidget(self.label_course)
        layout.addWidget(self.input_course)
        layout.addWidget(self.register_button)
        layout.addWidget(self.result_display)
        
        self.setLayout(layout)

    # Function to register a student
    def register_student(self):
        name = self.input_name.text()
        age = self.input_age.text()
        course = self.input_course.text()

        if name and age and course:
            student = Student(name, age, course)
            self.students.append(student)  # Store the student object
            self.result_display.append(student.get_details())  # Display student info

            # Clear input fields
            self.input_name.clear()
            self.input_age.clear()
            self.input_course.clear()
        else:
            self.result_display.append("⚠️ Please fill all fields!")

# Run the App
app = QApplication(sys.argv)
window = StudentApp()
window.show()
app.exec()

📌 Explanation

1. Class Student (Encapsulation)
Stores name, age, and course with a method to return details.

2. Class StudentApp (GUI)
Inherits from QWidget (OOP: Inheritance) and creates input fields, buttons, and display areas.

3. register_student() Function
Collects input, creates Student objects, displays data, and clears inputs.

🔹 Features

  • ✅ Easy input fields for student details
  • ✅ Click button to register a student
  • ✅ List of registered students appears in a text box
  • ✅ OOP-based, extendable for more features

🔍 GUI Components Used

This PySide6 application builds a simple Student Registration System using various GUI elements. The main window is created using QWidget, with a vertical layout (QVBoxLayout) to arrange elements. Labels (QLabel) provide field names, while text input fields (QLineEdit) allow users to enter student details. A button (QPushButton) registers the student, displaying results in a read-only text area (QTextEdit). All elements are placed within a structured layout for an intuitive user interface.

📌 GUI Elements Explained

  • QApplication: The main application handler.
  • QWidget: The base class for the main window.
  • QVBoxLayout: Arranges widgets in a vertical order.
  • QLabel: Displays labels for Name, Age, and Course.
  • QLineEdit: Text input fields for user input.
  • QPushButton: A button to trigger student registration.
  • QTextEdit: Displays registered students and messages.

🚀 How the GUI Works (Step-by-Step)

  1. The user enters Name, Age, and Course in QLineEdit fields.
  2. Clicking the Register Student button triggers register_student().
  3. A new Student object is created and stored in a list.
  4. The student's details are displayed in the QTextEdit area.
  5. Input fields are cleared for the next entry.
